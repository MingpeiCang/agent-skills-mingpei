# Writing Strategy

This file governs paragraph- and section-level rewriting logic. Read it when the task is not just sentence polishing but structural or argumentative improvement.

## Paper Type Identification

Before editing, determine the paper type. Different types have different narrative logic.

| Type | Reader's core question |
|------|----------------------|
| Research paper | Why does the phenomenon matter? What was done? What was found? What does it mean? |
| Methods paper | Does the method work? Is it reproducible? Is it better under fair comparison? |
| Hypothesis-based | Does the evidence establish or rule out a causal explanation? |
| Algorithmic/device | Does the system perform reliably and advantageously? |

Do not use one narrative logic for all paper types.

## Reader Workflow

Most readers follow a stable sequence:

1. **Relevance** — Is this relevant to me?
2. **Novelty** — What is new here?
3. **Trust** — Do I trust it?
4. **Reuse** — Can I reuse it?
5. **Meaning** — What does it mean, and where are the boundaries?

Polishing should help the paper answer these questions in this order.

## Hourglass Structure

Strong papers mirror an hourglass:

- **Introduction**: broad → narrow (open territory → gap → specific question → study)
- **Discussion/Conclusion**: narrow → broad (specific findings → implications → field-level significance)

If a paragraph or section violates this architecture, rebuild it before polishing wording.

## Claim-Evidence-Boundary

Every important scientific statement should have three parts:

1. **Claim**: what is being said
2. **Evidence**: what supports it
3. **Boundary**: where the claim stops, or what uncertainty remains

Typical failures:

- claim without evidence
- data without an explicit point
- implication without a scope condition
- correlation rewritten as mechanism

Repair these failures before polishing rhythm.

## Diagnosis Before Editing

Before rewriting, identify the main failure mode:

| Failure mode | Symptom |
|-------------|---------|
| Wrong paper type logic | Methods paper written as narrative, or research paper without gap |
| Missing gap or poor positioning | Introduction lists facts but never names what is missing |
| Claim without evidence | Strong assertion with no data, citation, or mechanism |
| Evidence without claim | Data presented but no explicit finding stated |
| Missing boundary | Implication stated without scope condition |
| Results/Discussion mixed | Interpretive language in Results, or raw data in Discussion |
| Weak title or abstract | Reader cannot tell what the paper does or why it matters |
| Sentence-level clutter only | Structure is sound but wording is bloated or imprecise |

Prioritize: `paper type → section job → paragraph logic → claim/evidence/boundary → sentence polish`

## Writing Order

The writing order is not the reading order. Organize around evidence and argumentative function.

### Research paper

1. Results
2. Introduction and Conclusion
3. Title
4. Discussion
5. Materials and Methods
6. Abstract

### Methods paper

1. Methods
2. Results
3. Introduction
4. Conclusion
5. Discussion
6. Abstract

## Section Responsibilities

### Introduction

Must answer:
1. What is already known?
2. What remains unresolved?
3. What exact question does this study ask?
4. How does the study address it?

Do not summarize Results or Conclusion here.

### Results

Results state what was observed. Provide:
- object or system
- condition
- quantitative support
- direct result

Do not turn Results into Discussion. Results answer "what happened", not "what it means".

### Discussion

Discussion explains what the findings mean:
- how the work fits the broader field
- what has been added to understanding
- which earlier work is being supported, revised, or complicated
- which explanations are plausible
- which limitations constrain interpretation

Discussion is the natural home for hedging.

### Conclusion

Three-part close:
1. Restate the central contribution
2. Summarize the decisive evidence
3. State the implication with a boundary

Do not introduce new data. Always run an overclaim check here.

### Abstract

The abstract is a mini-paper:
`context/problem → gap/objective → approach → key results → implication`

### Title

A strong title:
- tells the reader what to expect
- avoids unnecessary technical language
- is easy to search
- is substantiated by data
- creates curiosity without sacrificing credibility

Target: ≤ 75 characters including spaces.

## Overclaim Control

Watch for and soften:

| Flag word | Safer replacement |
|-----------|------------------|
| prove | show, demonstrate, suggest |
| conclusively | strongly, consistently |
| unprecedented | to our knowledge, rarely reported |
| best | among the strongest |
| unqualified first | to our knowledge, the first to |
| superior | improved, enhanced |
| always | in all tested cases |
| never | in no case examined |

Good Nature prose can be ambitious, but ambition must be attached to evidence, scale, and boundary.

## Citation as Positioning

Citation is not just formatting. It tells the reader how the current work stands relative to earlier work.

Categories:
- **support**: prior work supports the premise
- **borrow**: current work adopts a method or framework
- **contrast**: current work differs in result or interpretation
- **reuse**: material, data, code come from elsewhere

Always cite the source actually read and verified.

## Fairness to Earlier Work

Do not manufacture novelty by flattening previous studies into a weak baseline. Prefer:

- `Although previous studies showed ..., their performance in ... remains unclear.`
- `Earlier work established ..., but did not address ...`

This preserves intellectual honesty while making the gap explicit.

## Chinese-to-English Mode

When the source is Chinese or strongly Chinese-influenced English:

1. Extract core propositions first — do not translate clause-by-clause
2. Reconstruct explicit logical links: contrast, cause, implication, limitation
3. Verify terminology, causality, hedging, and disciplinary nuance
4. Keep key technical terms stable
5. Remove Chinese discourse patterns (e.g., excessive `With the development of...`, `In recent years, more and more...`)

## Paragraph Linking

Avoid repetitive `This suggests ...` openings. Prefer:

- restate the noun: `Such heterogeneity ...`
- definite noun phrase: `The resulting gradient ...`
- participial summary: `Taken together, ...`
- zero-connective progression when logic is already obvious

Limit demonstrative-led openings. One per paragraph is usually enough.
